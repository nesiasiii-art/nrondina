# Merge Duplicate Issues from Different Cases

**Target audience:** All (COMPANY) product support agents

When you encounter multiple users who report the same issue, you can merge these issues into a single case. This article explains the merged case process for duplicate issues.

The agent who has most recently worked on a specific customer issue must:
- Properly categorize and update all cases under the same issue.
- Mark subsequent cases as duplicates of the original.
- Avoid confusing a new issue, that may seem similar in some aspects, as a duplicate.

## Find Duplicate Issues
Use the following documentation on how to find duplicate issues:
- If a user says they've previously reached out about the same issue: Refer to the Repeated Contact Guidelines.
- For duplicate escalations: Refer to the Duplicated Escalation Process.

## Mark Issues as Duplicates
During a troubleshooting session, on your Interactions tool:
1. On "Interactions History," find existing issues that are the same as the current issue.
2. Add a relevant category to the duplicate issue.
3. If the user accepts the troubleshooting steps or resources you provided, transfer them to the Customer Satisfaction Survey.
	- **Through chat:**
		1. At the chatbox, click **Close**.
		2. At the end of the session, the user is automatically transferred to the survey.
	- **Through call:** Always end calls with an offer to engage with the survey.
		- To learn more, refer to the "Transfer a Caller to Customer Satisfaction Survey" process.
4. Click **Duplicate**. This opens a pop-up menu with the following options:
	- **Reason for marking as duplicate:** From the dropdown, select the appropriate item.
	- **Assign parent case to me:** Make sure that this box is checked.
5. Click **Mark as duplicate**. 
6. In the parent case, mark the issue as "Resolved."
